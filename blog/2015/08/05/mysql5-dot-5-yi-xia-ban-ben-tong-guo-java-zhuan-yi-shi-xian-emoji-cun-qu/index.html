MySQL5.5 以下版本通过 Java 转义实现 Emoji 存取

Aug 5th, 2015 10:41 pm

我不输出好文，我只是好文的搬运工。十分钟搞清字符集和字符编码带你走进 Unicode 和 UTF-8 的前世今生。总而言之，Unicode 是标准，UTF-8 是实现。

MySQL 篇

通常 MySQL 的默认字符集都会配置为 UTF-8，只支持单字符不超过 3 bytes 的存储1。常见的 Emoji 的 Unicode 编码位于 \u1F601--\u1F64F 区间，以 \u1F300 『🌀』举例，它的二进制有 17 位，无法用三字节的 UTF-8 编码表示2。如果通过 JDBC 将此字符尝试插入 MySQL 记录中，会得到以下异常

Java 篇

Java 存储字符的单元是 char，那么问题来了，16 位的 char 如何 handle 17 位的 Unicode 呢？

Java 的单个 char 无法 handle 🌀，喜闻乐见。那么 ⚡ 为什么可以通过编译，相信聪明如你，一定可以查到 ⚡ 的 Unicode 是 \u26A1，Tada
